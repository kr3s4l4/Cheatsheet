Cheatsheet de SQLite

SQLite es una base de datos ligera basada en archivos, muy común en aplicaciones móviles, escritorio, embebidas y también en retos CTF donde se proporciona un archivo .db.
Comandos básicos (cliente sqlite3)
Comando	Descripción
.tables	Lista todas las tablas
.schema	Muestra el esquema completo (CREATE TABLE)
.schema nombre_tabla	Muestra el CREATE de esa tabla
.dump	Exporta la base completa en SQL
.databases	Lista bases de datos conectadas (main, temp)
.output archivo.txt	Redirige salida a un archivo
.quit	Salir
.help	Ayuda
Consultas SQL útiles en hacking

Listar todas las tablas (sin usar .tables):
sql

SELECT name FROM sqlite_master WHERE type='table';

Listar todas las columnas de una tabla:
sql

SELECT sql FROM sqlite_master WHERE name='tabla' AND type='table';
-- O bien:
PRAGMA table_info(tabla);

Obtener versión de SQLite:
sql

SELECT sqlite_version();

Leer archivos del sistema (si la extensión readfile está habilitada):
sql

SELECT readfile('/etc/passwd');
-- Nota: no siempre disponible.

Escritura de archivos (si permisos):
sql

ATTACH DATABASE '/tmp/evil.db' AS pico; CREATE TABLE pico.exp (data TEXT); INSERT INTO pico.exp VALUES (load_extension('malicious'));

Comentarios en SQLite:
sql

-- Comentario de línea
/* Comentario bloque */

¿Por qué es útil en hacking?

    CTFs con descarga de archivos: muchas veces se da un archivo database.db y debes extraer la flag usando comandos de SQLite.

    Inyección SQL en web: si detectas que el back-end es SQLite, los payloads difieren de MySQL (por ejemplo, no hay UNION SELECT con information_schema, se usa sqlite_master).

    Pruebas de aplicaciones móviles: SQLite es la base por defecto en Android/iOS; extraer datos sensibles es clave.

    Funciones especiales: load_extension() puede llevar a RCE si está habilitado
